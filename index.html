Direct answer: what is a computer-use cache?

Reusable action traces

Super captures verified clicks, keystrokes, waits, and screen state from real computer use, then replays them deterministically for future runs.

Cheaper repeats

By reusing prior action traces, repeated workflows avoid re-planning and re-exploration overhead. You pay for learning once, then reuse.

Safer automation

Scopes, confirmations, and checkpoints keep assisted workflows aligned with app rules and user intent.

How Super builds and reuses the cache

  1. Observe once. The agent completes a task on a real computer while Super records a high‑fidelity action trace.
  2. Normalize. Actions are generalized (selectors, waits, retries) so small UI changes don’t break replays.
  3. Store. The trace is saved as a reusable computer-use cache entry with permissions and checkpoints.
  4. Replay. Future runs reuse the trace, skipping exploration and reducing cost.
